# New Ancestor Intake

What to do when you discover a new ancestor (from web research, a document, or a family member's information).

## Step 1: Verify Before Adding

Before adding a new person to the tree:

1. **Does this person match your family?** Verify name, dates, AND location all align with your existing tree.
2. **What is the source?** Record where this person was found.
3. **Is there a possible confusion?** Could this be a different person with the same name?

## Step 2: Update Family_Tree.md

Add the new person in the correct position:
- Under their parent (if known)
- In the correct family line section
- With all known facts (dates, places, relationships)
- With source citation
- Mark any unverified details with `(unverified)`

## Step 4: Update Supporting Files

After adding a new ancestor, check whether other files need updating:

| File | Update If... |
|---|---|
| Timeline.md | The person has any dated events |
| Open_Questions.md | The discovery resolves or creates a question |
| Data_Inventory.md | A new source document was found |
| Research_Log.md | The discovery came from a search (log the search) |
| Regions/[Region].md | The person is from a region with an existing region file |
| Witness_Network.md | The person was identified from a witness list |
| Unresolved_Persons.md | The discovery resolves a previously unresolved person |
| GEDCOM file | Always (if you maintain one) |

## Step 5: Check for Cascading Updates

A new ancestor can trigger cascading changes:
- If the new person is a parent, update the child's person file
- If the new person is a sibling, check whether their records mention shared parents
- If the discovery changes a date or name, run a cross-reference audit on affected files
